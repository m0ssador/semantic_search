# Описание алгоритма работы

## Предварительная обработка текста

Нормализуем названия для устранения вариативности:

*   приводим к нижнему регистру;
*   заменяем синонимы единиц измерения (" -> дюйм, gb -> гб, );
*   удаляем лишние пробелы и пунктуацию;
*   переводим цвета и категории (blue -> синий, смартфон -> smartphone)

>[!note]: 
>Вместо перевода можно более универсально удалить всю кириллицу, но есть риск потерять важные данные для сравнения, что повысит риск ложного срабатывания на дубликаты.

## Расчёт схожести

Используем комбинацию метрик:

*   **Levenshtein distance** (расстояние Левенштейна) для учёта опечаток;
*   **Jaccard similarity** для сравнения наборов слов;
*   **TF-IDF + косинусное сходство** для семантической близости.

Итоговая схожесть — комбинация из трех метрик с заданными весами (веса выставлены вручную по малой выборке).

## Фильтрация по порогу

Полученные пары фильтруются относительно выставленного порога **SIMILARITY_THRESHOLD** (по умолчанию **0,8**)

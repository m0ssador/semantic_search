# Итоговое задание по предмету Алгоритмы и структуры данных МФТИ

## Вариант 3: Нечёткий поиск, выявление дублирующихся значений

Крупный маркетплейс столкнулся с проблемой дублирования товарных предложений. Разные продавцы добавляют один и тот же товар, но в его описании могут быть опечатки, различные сокращения, синонимы, изменённый порядок слов или дополнительные характеристики. Это ухудшает пользовательский опыт и мешает системе корректно ранжировать предложения. Необходимо разработать алгоритм, который для каждого нового товара будет находить в каталоге потенциальные дубликаты.

### Описание входных данных

*   В файле catalog.txt дан существующий каталог товаров. Каждая строка представляет собой уникальный идентификатор товара и его название, разделенные табуляцией.
    
    ID_1\tНазвание товара 1

    ID_2\tНазвание товара 2

    …

*   Файл new_items.txt содержит список новых товаров, которые нужно проверить на дубликаты. Формат аналогичен.
    
    ID_N1\tНазвание нового товара 1

    ID_N2\tНазвание нового товара 2

    …

*   Параметры алгоритма (задаются в коде или конфигурации):

    SIMILARITY_THRESHOLD (порог схожести): число от 0 до 1. Значение 0.8 означает, что товары считаются дубликатами при 80% схожести.

    USE_PREPROCESSING (флаг предобработки): включить/выключить нормализацию текста.

### Пример входных данных:

➔ catalog.txt:

    1001 Смартфон Xiaomi Redmi Note 12 Pro 8/256GB синий

    1002 Телефон Huawei P60 Pro 12/512GB черный

    1003 Планшет Irbis TX97 10.1" 4/64GB

    1004 Xiaomi Robot Vacuum Cleaner S10+

    1005 HUAWEI Watch GT 4 46mm

➔ new_items.txt:

    2001 Xiaomi Redmi Note 12 Pro 8/256 Синий

    2002 Huawei P60 Pro 12/512 Black

    2003 Планшет IRBIS TX97 10.1 дюйм 4/64 ГБ

    2004 Смартфон Xiaomi Redmi 13C 4/128

    2005 Робот-пылесос Xiaomi S10+

➔ Параметры:

    SIMILARITY_THRESHOLD = 0.8

    USE_PREPROCESSING = True

### Описание выходных данных (в формате JSON)

В результате должен быть получен файлduplicates.json, где для каждого нового товара приведён список найденных дубликатов из каталога с указанием коэффициента схожести.

Например,

```json
{
  "2001": [
    {"catalog_id": "1001", "similarity_score": 0.92}
  ],
  "2002": [
    {"catalog_id": "1002", "similarity_score": 0.87}
  ],
  "2003": [
    {"catalog_id": "1003", "similarity_score": 0.89}
  ],
  "2004": [],
  "2005": [
    {"catalog_id": "1004", "similarity_score": 0.85}
  ]
}
```
